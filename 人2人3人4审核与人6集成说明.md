# 人2/人3/人4交付审核与人6集成说明

## 审核结论

### 人2：真值表模块

最新更正后实际审核的是 `truth_table.zip`。此前的 `DeliverLetters.zip` 是误发包，不再作为人2交付结论依据。

`truth_table.zip` 中包含：

- `truth_table.cpp`
- `truth_table.exe`

审核结果：

- `truth_table.cpp` 与人2任务基本匹配，能够调用统一的 `parser.h/parser.cpp` 解析公式。
- 源码实现了变量提取、赋值枚举、AST 递归求值和 JSON 真值表输出。
- 原包自带的 `truth_table.exe` 在当前环境直接运行时未得到有效输出，判断为可执行文件运行环境或编译兼容性问题；用当前工程重新编译该源码后，`A<->B` 和 `(A∧B)→C` 均可输出合法 JSON。

已在集成交付中统一保留并验证：

- `truth_table.cpp`
- `truth_table.exe`

功能：

- 调用统一 `parser.h/parser.cpp` 解析公式。
- 递归提取变量并按字母序排序。
- 枚举所有赋值组合。
- 输出接口规范要求的 JSON 真值表。

### 人3：CNF 模块

原始交付包 `LogicCNF_人3交付包.zip` 包含 CNF 转换思路和工程文件，但联调时需要统一变量编号、JSON 输出和命令行行为。

已补齐并统一：

- `cnf.cpp`
- `cnf.exe`

功能：

- 调用统一解析器解析公式。
- 输出 `{"clauses":[...],"var_map":{...}}`。
- `var_map` 按变量字母序从 1 开始编号。

### 人4：DPLL 模块

原始交付包 `dpll.vcxproj.zip` 包含 DPLL 工程和 exe。审核时发现两个关键问题：

1. Windows PowerShell 管道输入 JSON 时可能以 UTF-16 形式传入，原程序无法正确解析。
2. 原 DPLL 在“没有单子句”的情况下会误判为冲突，导致部分可满足 CNF 被判为 UNSAT。

已修复并统一：

- `dpll.cpp`
- `dpll.exe`

功能：

- 支持 stdin 输入 CNF JSON。
- 支持 SAT / UNSAT 输出。
- SAT 时 assignment 覆盖 `var_map` 中所有变量。

## 人6完成内容

新增：

- `puzzles.py`

### 谜题建模

`puzzles.py` 提供三个 SAT 谜题：

1. 三人排课
2. 4x4 数独
3. 宠物与颜色逻辑谜题

每个谜题都输出 CNF JSON，并提供 DPLL 结果解码函数。

### Streamlit 可视化

`app.py` 提供两个视图：

1. 谜题求解：选择谜题，展示 CNF、DPLL 输出和解码结果。
2. 公式测试：输入命题逻辑公式，展示 AST、真值表、CNF 和 DPLL。

启动：

```powershell
streamlit run app.py
```
